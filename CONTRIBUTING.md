# 🤝 Вклад в FavoriteCLI

Спасибо за интерес к участию в проекте FavoriteCLI!

## 📋 Как внести вклад

### 1. Отчеты об ошибках

Перед созданием issue:
- ✅ Проверьте, что проблема ещё не зафиксирована
- ✅ Укажите версию проекта (`favorite --version`)
- ✅ Опишите шаги воспроизведения
- ✅ Приложите логи (если уместно)

Пример формата:
```
**Описание:** [Краткое описание проблемы]
**Версия:** [версия]
**Шаги:** 
1. ...
2. ...
**Ожидаемое поведение:** ...
**Фактическое поведение:** ...
```

### 2. Предложение новых функций

Вместе опишите:
- Зачем это нужно?
- Как это улучшит проект?
- Есть ли альтернативные решения?

### 3. Pull Request'ы

1. Fork репозиторий
2. Создайте ветку для feature (`git checkout -b feature/amazing-thing`)
3. commit'ите изменения (`git commit -m 'Add amazing feature'`)
4. push'ните в ветку (`git push origin feature/amazing-thing`)
5. Откройте Pull Request

**Правила кода:**
- Следуйте PEP 8
- Пишите тесты для нового кода
- Обновляйте документацию
- Пропускайте CI/CD проверки

### 4. Стиль кода

- Используйте Python 3.8+
- Docstrings для всех функций (Google style)
- Логические import'ы вверху файла
- Обработка исключений везде где это уместно

## 🛠️ Разработка

Установка для разработки:

```bash
git clone https://github.com/yourusername/FavoriteCLI.git
cd FavoriteCLI
pip install -r requirements.txt -r requirements-dev.txt
```

Запуск в режиме разработки:

```bash
python -m favorite.app
```

Тесты:

```bash
pytest favorite/tests/ -v
```
